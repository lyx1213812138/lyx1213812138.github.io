  1. 外部 js 和 css 文件一般怎么引入？在什么位置引入？为什么？

    在网页开发中，外部 CSS 文件通常通过 <link> 标签引入，并放在 HTML 的 <head> 部分，这样可以让浏览器在解析页面时提前加载和应用样式，避免页面内容先显示为无样式状态，同时优化页面的渲染性能。而外部 JavaScript 文件则推荐放在 <body> 的底部，因为 JavaScript 文件可能会阻塞页面加载，放在底部可以确保页面的 HTML 内容先加载和渲染，从而提升用户体验。不过，如果需要在 <head> 中引入 JavaScript 文件，可以使用 async 或 defer 属性来避免阻塞页面解析，实现异步加载或延迟执行。这种引入方式的选择主要是为了优化页面加载性能和确保页面内容能够快速呈现给用户。

  2. 你了解 HTML 头部（<head>）吗？说说里面可以放哪些东西，有什么用

    HTML 的 <head> 部分是文档头部的容器，位于 <html> 标签内，紧跟在 <head> 标签之后。它主要用于包含元数据、链接外部资源以及定义文档的全局属性。虽然 <head> 部分的内容不会直接显示在网页上，但它对页面的加载、性能、SEO（搜索引擎优化）以及用户体验都有着重要的影响。

  3. <meta> 标签

    • 字符集声明：<meta charset="UTF-8"> 是必不可少的，它告诉浏览器页面使用的是哪种字符编码（通常是 UTF-8），确保页面能正确显示各种字符。

    • 视口设置：<meta name="viewport" content="width=device-width, initial-scale=1.0"> 用于优化页面在移动设备上的显示效果，确保页面能够自适应不同屏幕尺寸。

    • SEO 元数据：通过 <meta name="description" content="页面描述"> 和 <meta name="keywords" content="关键词"> 等标签，可以为搜索引擎提供页面的描述和关键词信息，有助于提升页面的搜索排名。

  4. <title> 标签

    • 定义网页的标题，显示在浏览器的标签页上。它对用户体验和 SEO 都非常重要，因为它是用户在搜索结果中看到的第一个内容，也是收藏夹中显示的标题。

  5. <link> 标签

    • 用于链接外部资源，最常见的用途是引入外部 CSS 文件，如 <link rel="stylesheet" href="styles.css">。此外，还可以用于定义网站图标（<link rel="icon" href="favicon.ico">）或预加载关键资源（<link rel="preload">）。

  6. 你知道 preload 和 prefetch 吗？它们分别有什么用？

preload 是一种声明式的资源加载机制，用于告诉浏览器尽快加载当前页面的关键资源。它具有较高的优先级，通常用于加载页面渲染所需的资源，如 CSS、JavaScript 或字体文件。 prefetch 是一种在浏览器空闲时提前加载未来可能需要的资源的机制。它不会阻塞页面的加载，而是利用空闲时间在后台下载资源。
